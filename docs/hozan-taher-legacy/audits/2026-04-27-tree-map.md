## features/outreach/campaigns

Core campaign execution: run-once-per-tick orchestration, SMTP sender with anti-trace relay integration, template rendering, humanization.

### campaign/
**Tick-driven scheduler and per-contact runner. Compliance gates: two-table suppression filter, domain rotation, holding clusters, timezone-aware send windows.**

```
campaign/
├── runner.go (RunCampaign, NewRunner, NewReadOnlyRunner, CreateCampaign)
│   ├── public: RunCampaign(ctx, campaignID) — per-tick entrypoint
│   ├── public: CreateCampaign(ctx, name, description, steps, filter) → campaignID
│   ├── public: List, Get, SetStatus, Stats, EstimateEnrollment
│   ├── private: suppressionFilterFor(col) — canonical two-table NOT-IN clause
│   ├── private: enrollContacts(ctx, campaignID, filter) — SELECT... ON CONFLICT DO NOTHING
│   ├── private: regionToTimezone(region) — "SK" → Europe/Bratislava; else Europe/Prague
│   ├── private: buildUnsubURL(campaignID, contactID, email) → HMAC-SHA256 token
│   ├── gates: RunCampaign enforces
│   │  ├── calendar.IsSendableDay(now) — Czech weekends + holidays (SKIP_CALENDAR_CHECK=1 bypass)
│   │  ├── EmailStatusAllowed(emailStatus) — whitelist (valid, unknown, risky)
│   │  ├── HoldingClusterCap (seenParentICO[ico]++) — one per parent_ico per tick
│   │  ├── MaxPerDomainPerTick (seenDomain[domain]++) — domain rotation
│   │  ├── MaxPerDomainDay (domainDayCount cache) — persistent 24h limit per domain
│   │  ├── calendar.InSendWindow(now, tz) — Mon-Fri 08-17h in recipient TZ
│   │  └── contactStatus NOT IN (bounced, unsubscribed, opted_out, paused_human, …)
│   │      UNION suppressionFilterFor("c.email") — must mirror scheduler_postgres.go ListRunningCampaigns
│   ├── CAS: current_step = $oldStep predicate guards concurrent advance (H1 2026-04-21)
│   ├── audit: audit.Log outside tx, one row per campaign tick (BF-E6 contract)
│   └── status re-check: mid-tick every statusCheckEvery enqueues (lazy pause acknowledgement)
├── scheduler.go (NewScheduler, Start, Tick)
│   ├── public: Scheduler.Tick(ctx) — fetch running campaigns, acquire lock, RunCampaign
│   ├── public: Scheduler.Start(ctx, interval) — loop until ctx.Done()
│   ├── private: tick() → ListRunningCampaigns → runOne(id) for each
│   ├── private: runOne(id) — TryAdvisoryLock → RunCampaign → ReleaseAdvisoryLock
│   ├── panic recovery: per-campaign; one panic must not abort siblings
│   ├── status filter: mirrors runner.go (draft + active + running)
│   ├── interval: SCHEDULER_INTERVAL_SEC env or 60s default
│   └── dispatch: concurrent-safe via pg_try_advisory_lock (BF-E4)
├── scheduler_postgres.go (PostgresAdvisoryLocker, PostgresSchedulerDB)
│   ├── public: ListRunningCampaigns(ctx) → []{ID, Status} for status in (draft, active, running)
│   ├── public: TryAdvisoryLock(ctx, id) — pg_try_advisory_lock(id) non-blocking
│   ├── public: ReleaseAdvisoryLock(ctx, id) — pg_advisory_unlock(id) on defer cleanup
│   ├── audit: campaign_lock_audit table (migration 007) for stale-lock detection
│   ├── StaleLockCheck(ttl) → []LockRecord with created_at < now() - ttl
│   └── lock IDs: 2^31 space (safe per Postgres docs; collision: 1 in 2B campaigns)
├── gate.go (EmailStatusAllowed, HoldingClusterCap, MaxPerDomainPerTick, MaxPerDomainDay constants)
│   ├── const: HoldingClusterCap = 5 — max contacts per parent_ico per tick
│   ├── const: MaxPerDomainPerTick = 3 — domain rotation within single tick
│   ├── const: MaxPerDomainDay = 50 — persistent 24h cap across all mailboxes
│   └── allowlist: EmailStatusAllowed checks WHITELIST (valid, unknown, risky)
├── batch.go (BatchInsert, BatchSize)
│   └── DRY batch INSERT generator (unused in current runner; candidate for refactor)
├── dedup.go (DeduplicateContacts)
│   └── Set-based dedup before enrollment (legacy; contacts table UNIQUE already enforces)
```

### sender/
**SMTP engine with per-mailbox circuit breaker, greylisting exponential backoff, anti-trace-relay client, per-domain bounce tracking.**

```
sender/
├── engine.go (Engine, NewEngine, Run, Enqueue, pickMailbox)
│   ├── public: Engine.Enqueue(req SendRequest) — nonblocking append to queue
│   ├── public: Engine.Run(ctx) — consume queue, pickMailbox, call antitrace.Submit
│   ├── public: Engine.WithPreSendHook(hook), WithAntiTrace(client), WithWarmupLimiter(l)
│   ├── public: Engine.ResetMailboxBreaker(addr) — half-open state from watchdog
│   ├── state: mu sync.Mutex guards queue, sentCounts, domainCounts, bounceCount
│   ├── state: currentIdx int — FIFO position in queue (not-reentrant)
│   ├── state: circuitOpen bool — global circuit breaker (totalBounceRate > Threshold)
│   ├── state: domainCircuitOpen[domain] → time.Time — per-domain breaker
│   ├── state: mailboxCooldownUntil[addr] → time.Time — per-mailbox connection failures
│   ├── state: mailboxConsecutiveFails[addr] int — threshold = 3, cooldown = 30m
│   ├── const: mailboxFailThreshold = 3 — dial/TLS/auth failures trigger cooldown
│   ├── const: mailboxCooldown = 30 * time.Minute
│   ├── concurrency: pickMailbox not goroutine-safe; Run serializes
│   └── registry: optional mailbox.Backpressure wiring (D2.3) for active-set filtering
├── antitrace.go (AntiTraceClient, Submit, typed errors)
│   ├── public: AntiTraceClient.Submit(ctx, req) → SendResult
│   ├── public: errors ErrAntiTraceRateLimited, ErrAntiTraceTransport, ErrAntiTraceAuth (typed)
│   ├── endpoint: relay-service SMTP submit at /submit (hardcoded URL or env RELAY_URL)
│   ├── credentials: passed via SMTPHost/SMTPPort/SMTPUsername/SMTPPassword in req
│   ├── from: relay chooses from relay's pool, not client's mailbox (anti-trace isolation)
│   └── all errors wrapped with %w so errors.Is() chains work
├── backoff.go (BackoffClassifier, SMTPClass, RetryFor, ExponentialBackoff)
│   ├── public: ClassifyError(smtpCode, text) → SMTPClass (Transient, Permanent, Unrecognized)
│   ├── public: RetryFor(class, attempt) → duration or 0 (permanent)
│   ├── classifiers:
│   │  ├── 4xx → Transient: 1s, 8s, 64s, …
│   │  ├── 5xx → Transient: 1s, 8s, 64s, …
│   │  ├── 550 / 551 → Permanent: 0
│   │  ├── connection failures → Transient: backoff
│   │  └── unmatched codes → Unrecognized: fallback
│   └── poissonDelay(lambda) — jitter for thundering herd
├── probe_adapter.go (ProbeAdapterClient, SendRequest forwarding to relay probe endpoints)
│   ├── public: ProbeAdapter.ProbeAuth(ctx, creds) → AuthResult (valid/invalid/unknown)
│   ├── public: ProbeAdapter.ProbeRelay(ctx, relay) → RelayHealth (latency, TLS version)
│   ├── endpoint: relay-service /probe (BFF-side wiring)
│   └── fallback: returns unknown on transient errors (fail-open)
├── trace.go (Trace event logging — deprecated, migrated to sender/engine metrics)
│   └── legacy: EventTrace struct for audit; modern version goes to Sentry
```

2. **server.js:75 .env parsing** — `split('=')` with no quoted-string handling allows `KEY=val=ue` → value becomes `ue`. Worse: no validation of key names before setting `process.env[k]`. An attacker controlling `.env` (e.g., via git diff) could inject `NODE_ENV=production` to flip prod mode or `DATABASE_URL=evil.com` to redirect DB. Mitigation: use `dotenv` module with schema validation.

3. **campaigns/sender/engine.go:pickMailbox — no goroutine lock** — Engine.mu guards the queue, but pickMailbox (called from Run) is not re-entrant. If PreSendHook spawns a goroutine that calls Run again (unlikely but possible with async webhooks), concurrent writes to sentCounts/domainCounts/mailboxCooldownUntil corrupt state. Mitigation: make pickMailbox safe under concurrent calls or document serialization contract.

4. **campaign/runner.go:344 Render with contactID** — runner calls r.content.Render(step.TemplateName, vars, contactID, step.Step). If TemplateName is user-controlled (it is: campaign.sequence_config[step].template), template injection is possible. An attacker creating a campaign could inject `{{.UnsubURL}}` twice, `{{range .}}`, or other Go text/template tricks to leak vars. Later code assumes successful DB writes (e.g., endpoint_hits flush). If pool.query fails, the app silently continues with stale metrics. ## Top 6 Concurrency Hazards

1. **Engine.sentCounts map** — Global shared state, RWMutex guards it. But pickMailbox reads it without lock; only Run holds the lock. If another goroutine (e.g., recalc callback) calls pickMailbox concurrently, data race on sentCounts[mailbox]. Mitigation: push lock guard into pickMailbox itself, not caller.

2. **domainDayCount lazy cache** — runner.go line 286–305: domainDayCount is a local map. Two goroutines racing on different contacts with same domain can both miss cache, issuing concurrent DB queries for the same count. Not a safety issue (queries are read-only), but efficiency miss. Mitigation: use sync.Once per domain or atomic.LoadInt64.

3. **goroutine leak in runner recalc** — runner.go line 420: `go func()` spawns recalc without waiting. If context is cancelled mid-tick, the goroutine may panic + defer log, but it's not awaited. On daemon restart, in-flight recalcs orphaned. Mitigation: track spawned goroutines in WaitGroup, cancel before exiting.

4. **mailbox.Registry.RecordBounce async callback** — sender/engine.go calls goRegistryCall (panic-recovered), but no guarantee the callback fires before next tick. If a bounce is recorded after the engine already called pickMailbox for next send, stale state. Mitigation: make registry updates synchronous, not async.

5. **Sentry async capture** — telemetry/sentry.go CaptureException is fire-and-forget. If service crashes immediately, Sentry event may not flush. Mitigation: use BeforeSend hook to prioritize critical errors, increase timeout to 5s.

6. **mailbox_alerts race** — bounce/flip.go MaybeAutoHold writes to mailbox_alerts + updates outreach_mailboxes.status. If two bounces land concurrently, both might issue UPDATE status → bounce_hold, and both write to mailbox_alerts. Alert dedup at 5m window, but two INSERTs is redundant. Mitigation: use UPDATE outreach_mailboxes SET status = bounce_hold WHERE status = active (CAS).

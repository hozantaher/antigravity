# UX v2.0 — Claude.ai aesthetic, parallel rebuild

**Status:** Active
**Datum:** 2026-05-31
**Trigger:** Operator pivot — split-pane refit of v1 surfaces hit a UX ceiling
("ux/ui je katastrofické"). Decision: stop patching v1 layout, generate a
clean v2.0 design language and rebuild surfaces on it, parallel and additive,
so v1 keeps running untouched during the migration.

---

## North star

Perfect, simple, FUNCTIONAL UX in the **real Claude.ai look** — warm cream
surfaces, warm near-black ink, terracotta accent, serif display + Inter body,
generous whitespace, soft radii, calm loading/empty/error states, never a
false/misleading `0`. Calm daily triage of the whole pipeline (vozidla ·
firmy · kontakty · outreach) in one flow.

This reverses the May "Signal Desktop" pivot (`tokens-claude.css` is
Signal-clinical despite its filename). Operator explicitly chose the warm
Claude.ai direction on 2026-05-31 via AskUserQuestion ("Skutečný Claude.ai
look"). See memory `feedback_claude_style_means_anthropic` (updated to record
this reversal).

## Why parallel, not in-place

- v1 is live and operator-critical every day. An in-place reskin risks the
  whole dashboard on every commit.
- v2 lives at `/v2`, scoped under `.v2-app`, reading only `--v2-*` tokens. v1
  reads `--c-*` / `--accent` and is 100% unaffected. Breadth stays 14/14.
- Surfaces graft onto the v2 shell one phase at a time. When v2 reaches
  parity, `/v2` becomes the default and v1 is retired.

## Architecture

```
src/v2/
  styles/tokens-v2.css   design system — light default + dark parity, .v2-app scoped
  AppShellV2.jsx         calm sidebar (232px) + minimal topbar + <Outlet/>
  pages/
    V2Home.jsx           serif-hero landing + surface tiles
    V2Placeholder.jsx    calm "připravujeme" state per not-yet-rebuilt surface
```

Route: `/v2` is a **sibling** to the v1 pathless `<Layout/>` route in
`main.jsx` (static `/v2` segment outranks v1's `*` catch-all). Nested routes:
`index` (Přehled), `odpovedi`, `vozidla`, `firmy`, `kontakty`, `kampane`.

Theme: `data-theme` on `.v2-app` (light | dark), persisted to `localStorage`
key `v2Theme`. Both themes are full parity — dark is "Claude night" warm dark,
not an inverted light.

## Guardrails carried into v2

- Every surface needs a Playwright smoke (goto + visible headline + no
  console error) in the same PR (`feedback_playwright_smoke_required`).
- No false `0` — calm empty/loading/error states everywhere.
- No PII inline; egress only via anti-trace-relay; no campaign send without
  explicit consent; schema-verify before SQL.
- "Leady JSOU vozidla" — pipeline = Vozidla inventory, not a separate leads
  funnel.

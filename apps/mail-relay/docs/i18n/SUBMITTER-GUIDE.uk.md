# Посібник відправника

Як безпечно надіслати повідомлення через анти-трейс реле. Три кроки, жодних слідів на пристрої.

---

## Що потрібно

1. **Парольна фраза** -- фраза для запам'ятовування (4+ слів), спільна з отримувачем
2. **Ключ отримувача** -- 64-символьний hex-код (публічний ключ отримувача)
3. **Адреса реле** -- `.onion` URL сервера реле

Ці три дані вам має передати ваш контакт **безпечним каналом** (особисто, зашифрованим чатом, на папері). Ніколи електронною поштою чи телефоном.

---

## Крок 1: Підготуй середовище

### Найбезпечніше: Tails USB

1. Завантаж [Tails](https://tails.net) на іншому комп'ютері
2. Запиши на USB-флешку
3. Завантажся з USB (при старті тримай клавішу для boot menu)
4. Tails автоматично маршрутизує все через Tor
5. Після вимкнення не залишається жодних слідів

### Альтернатива: Tor Browser

1. Завантаж [Tor Browser](https://www.torproject.org)
2. Запусти та зачекай на підключення до мережі Tor
3. Решту виконай у терміналі з `torsocks`

---

## Крок 2: Завантаж submit binary

Завантаж файл для своєї системи:

| Система | Файл |
|---------|------|
| Linux (Intel/AMD) | `submit` з `linux-amd64/` |
| Linux (ARM, Raspberry Pi) | `submit` з `linux-arm64/` |
| macOS (Intel) | `submit` з `darwin-amd64/` |
| macOS (Apple Silicon) | `submit` з `darwin-arm64/` |

```bash
# На Tails/Linux:
chmod +x submit
```

### Перевір контрольну суму

```bash
sha256sum submit
# Порівняй з файлом SHA256SUMS від твого контакту
```

## Крок 3: Надішли повідомлення

```bash
echo "твоя-таємна-фраза" | ./submit \
  --relay https://xxxxxxxxxxxxxxxx.onion \
  --recipient-key abcdef0123456789abcdef0123456789abcdef0123456789abcdef0123456789 \
  --message "Твоє повідомлення тут"
```

Побачиш:
```
Passphrase:
Submitted successfully.
```

**Готово.** Повідомлення зашифроване та надіслане. Програма не залишила жодних файлів.

---

## Що відбувається за лаштунками

1. З твоєї фрази виводиться ключ шифрування (28 секунд обчислення -- це захист)
2. Повідомлення шифрується ключем отримувача (ніхто інший не може його прочитати)
3. Повідомлення доповнюється до стандартного розміру (щоб його не можна було розпізнати)
4. Надсилається до "мертвої поштової скриньки" на сервері реле
5. Отримувач забирає повідомлення з тієї ж скриньки (знає ту саму фразу)
6. Всі ключі та дані стираються з пам'яті
7. Програма завершується

---

## Правила безпеки

### Роби

- Використовуй Tails або хоча б Tor Browser
- Надсилай з публічної WiFi (кав'ярня, бібліотека)
- Запам'ятай фразу -- не записуй її нікуди
- Видали програму після надсилання (`rm submit`)
- Кожне повідомлення надсилай з іншого місця (якщо можливо)

### Не роби

- Не надсилай з домашньої WiFi
- Не використовуй телефон з SIM на твоє ім'я
- Не зберігай фразу в телефоні, нотатках чи хмарі
- Не залишай програму на комп'ютері довше ніж потрібно
- Не кажи нікому, що використовуєш цю систему
- Не надсилай однакове повідомлення кількома каналами (кореляція)

---

## Ситуація примусу (duress)

Якщо хтось змушує тебе показати систему:

1. Введи **іншу фразу** (не справжню)
2. Система покаже: "Authentication failed. Please try again."
3. Всі дані автоматично зітруться з пам'яті
4. Для спостерігача це виглядає як помилка друку -- неможливо відрізнити від примусу

---

## Часті запитання

**Скільки часу займає надсилання?**
~30 секунд (більшість -- виведення ключа з фрази -- це навмисно).

**Що якщо введу неправильну фразу?**
Покаже "Authentication failed." Нічого не надішлеться, нічого не збережеться.

**Чи може оператор реле прочитати повідомлення?**
Ні. Повідомлення зашифроване ключем отримувача. Реле бачить лише зашифрований блоб.

**Чи може хтось дізнатися, кому я пишу?**
Реле не знає, хто ти (псевдонім) і куди йде повідомлення (мертва скринька).

**Що якщо реле конфісковане?**
Повідомлення зашифровані. Без ключа отримувача їх не можна прочитати.

## Для отримувача: Отримання повідомлень

Отримувач використовує файл `receive` з тією ж спільною фразою:

### Показати публічний ключ (один раз, поділитися з відправником)

```bash
echo "спільна-фраза" | ./receive --show-key
```

### Отримати повідомлення

```bash
echo "спільна-фраза" | ./receive --relay https://xxxxxxxx.onion
```

### Важливо

- Повідомлення можна отримати **лише один раз** -- після отримання вони видаляються
- Фразу ділись **особисто** або через зашифрований канал
- Публічний ключ достатньо поділитися один раз

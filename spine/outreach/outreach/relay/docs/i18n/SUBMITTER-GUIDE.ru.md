# Руководство отправителя

Как безопасно отправить сообщение через анти-трейс реле. Три шага, никаких следов на устройстве.

---

## Что нужно

1. **Парольная фраза** -- запоминаемая фраза (4+ слов), общая с получателем
2. **Ключ получателя** -- 64-символьный hex-код (публичный ключ получателя)
3. **Адрес реле** -- `.onion` URL сервера реле

Эти три данных вам должен передать ваш контакт **по безопасному каналу** (лично, в зашифрованном чате, на бумаге). Никогда по электронной почте или телефону.

---

## Шаг 1: Подготовь окружение

### Самое безопасное: Tails USB

1. Скачай [Tails](https://tails.net) на другом компьютере
2. Запиши на USB-флешку
3. Загрузись с USB (при старте зажми клавишу для boot menu)
4. Tails автоматически направляет весь трафик через Tor
5. После выключения не остается никаких следов

### Альтернатива: Tor Browser

1. Скачай [Tor Browser](https://www.torproject.org)
2. Запусти и дождись подключения к сети Tor
3. Остальное выполни в терминале с `torsocks`

---

## Шаг 2: Скачай submit binary

Скачай файл для своей системы:

| Система | Файл |
|---------|------|
| Linux (Intel/AMD) | `submit` из `linux-amd64/` |
| Linux (ARM, Raspberry Pi) | `submit` из `linux-arm64/` |
| macOS (Intel) | `submit` из `darwin-amd64/` |
| macOS (Apple Silicon) | `submit` из `darwin-arm64/` |

```bash
# На Tails/Linux:
chmod +x submit
```

### Проверь контрольную сумму

```bash
sha256sum submit
# Сравни с файлом SHA256SUMS от твоего контакта
```

## Шаг 3: Отправь сообщение

```bash
echo "твоя-секретная-фраза" | ./submit \
  --relay https://xxxxxxxxxxxxxxxx.onion \
  --recipient-key abcdef0123456789abcdef0123456789abcdef0123456789abcdef0123456789 \
  --message "Твое сообщение здесь"
```

Увидишь:
```
Passphrase:
Submitted successfully.
```

**Готово.** Сообщение зашифровано и отправлено. Программа не оставила никаких файлов.

---

## Что происходит за кулисами

1. Из твоей фразы выводится ключ шифрования (28 секунд вычислений -- это защита)
2. Сообщение шифруется ключом получателя (никто другой не может его прочитать)
3. Сообщение дополняется до стандартного размера (чтобы его нельзя было распознать)
4. Отправляется в «мертвый почтовый ящик» на сервере реле
5. Получатель забирает сообщение из того же ящика (знает ту же фразу)
6. Все ключи и данные стираются из памяти
7. Программа завершается

---

## Правила безопасности

### Делай

- Используй Tails или хотя бы Tor Browser
- Отправляй с публичного WiFi (кафе, библиотека)
- Запомни фразу -- не записывай её нигде
- Удали программу после отправки (`rm submit`)
- Каждое сообщение отправляй из другого места (если возможно)

### Не делай

- Не отправляй с домашнего WiFi
- Не используй телефон с SIM на твое имя
- Не сохраняй фразу в телефоне, заметках или облаке
- Не оставляй программу на компьютере дольше необходимого
- Не говори никому, что используешь эту систему
- Не отправляй одинаковое сообщение несколькими каналами (корреляция)

---

## Ситуация принуждения (duress)

Если тебя заставляют показать систему:

1. Введи **другую фразу** (не настоящую)
2. Система покажет: "Authentication failed. Please try again."
3. Все данные автоматически сотрутся из памяти
4. Для наблюдателя это выглядит как опечатка -- нельзя отличить от принуждения

## Часто задаваемые вопросы

**Сколько времени занимает отправка?**
~30 секунд (большая часть -- вывод ключа из фразы -- это намеренно).

**Что если введу неправильную фразу?**
Покажет "Authentication failed." Ничего не отправится, ничего не сохранится.

**Может ли оператор реле прочитать сообщение?**
Нет. Сообщение зашифровано ключом получателя. Реле видит только зашифрованный блоб.

**Может ли кто-то узнать, кому я пишу?**
Реле не знает, кто ты (псевдоним) и куда идет сообщение (мертвый ящик).

**Что если реле конфисковано?**
Сообщения зашифрованы. Без ключа получателя их нельзя прочитать.

## Для получателя: Получение сообщений

Получатель использует файл `receive` с той же общей фразой:

### Показать публичный ключ (один раз, поделиться с отправителем)

```bash
echo "общая-фраза" | ./receive --show-key
```

### Получить сообщения

```bash
echo "общая-фраза" | ./receive --relay https://xxxxxxxx.onion
```

### Важно

- Сообщения можно получить **только один раз** -- после получения они удаляются
- Фразу делись **лично** или через зашифрованный канал
- Публичный ключ достаточно поделиться один раз

## Cross-cutting rails

**Test discipline (every sprint):**
- TDD red→green→refactor for every feature
- ≥10 test cases per substantive change (HARD RULE per memory)
- Property/fuzz tests for any pure-function addition
- E2E lock for any new UI surface
- BFF contract for any new `/api/*` route

**Commit discipline:**
- Go test baseline verified per commit (`go test -count=1 ./...`)
- Same-count invariant for migration commits
- Adapter pattern for carves (preserve legacy tests zero-mod)

**Chat A/B signals:**
- `Needs-Tests: <modul> <popis>` for feature commits
- `Breaks-Contract: <api|event|schema>` for API shape changes
- `Resolves-Trailer: Needs-Tests: …` from Chat B

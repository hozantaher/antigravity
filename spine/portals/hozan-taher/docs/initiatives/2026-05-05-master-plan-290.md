## Risk register

**Riziko Tier-1 ne-completion před Day 7:** operator UX zůstane bolavé, ale launch path drží. Mitigation: prioritizovat Tier-1 položky podle pain frequency (denní > týdenní).

**Riziko regression během 290-fix avalanche:** spousta paralelních agentů → conflict pravděpodobnost. Mitigation: isolation:"worktree" + eager admin-merge + post-batch test verification před dalším batchem.

**Riziko scope creep Tier-3 → Tier-4:** "ještě jedna feature" syndrom. Mitigation: explicit cut-off Sprint-D end Month-3, deferred items do backlog.

**Riziko že 290 nezvládneme do 6 měsíců:** realistic — některé Tier-4 features (BIMI, multi-user) mohou se posunout. Mitigation: per Tier-5 bag co se neoplatí dělat, ostatní per quarter prioritization review.

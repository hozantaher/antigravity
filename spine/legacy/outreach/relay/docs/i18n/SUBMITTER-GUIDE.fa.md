# راهنمای فرستنده

نحوه ارسال امن پیام از طریق رله ضد ردیابی. سه مرحله، بدون هیچ ردی روی دستگاه.

---

## آنچه نیاز دارید

1. **عبارت عبور** -- جمله‌ای قابل حفظ (۴ کلمه یا بیشتر)، مشترک با گیرنده
2. **کلید گیرنده** -- کد هگزادسیمال ۶۴ کاراکتری (کلید عمومی گیرنده)
3. **آدرس رله** -- آدرس `.onion` سرور رله

این سه مورد باید توسط مخاطب شما **از طریق کانال امن** (حضوری، چت رمزنگاری‌شده، یا کاغذ) به شما تحویل داده شود. هرگز از طریق ایمیل یا تلفن ارسال نکنید.

---

## مرحله ۱: محیط را آماده کنید

### امن‌ترین: Tails USB

1. [Tails](https://tails.net) را روی کامپیوتر دیگری دانلود کنید
2. روی فلش USB بنویسید
3. از USB بوت کنید (هنگام شروع، کلید منوی بوت را نگه دارید)
4. Tails به‌طور خودکار تمام ترافیک را از طریق Tor هدایت می‌کند
5. پس از خاموش‌شدن هیچ ردی باقی نمی‌ماند

### جایگزین: مرورگر Tor

1. [Tor Browser](https://www.torproject.org) را دانلود کنید
2. اجرا کنید و منتظر اتصال به شبکه Tor بمانید
3. بقیه مراحل را در ترمینال با `torsocks` انجام دهید

---

## مرحله ۲: فایل submit را دانلود کنید

فایل اجرایی مناسب سیستم خود را دانلود کنید:

| سیستم | فایل |
|-------|------|
| Linux (Intel/AMD) | `submit` از `linux-amd64/` |
| Linux (ARM, Raspberry Pi) | `submit` از `linux-arm64/` |
| macOS (Intel) | `submit` از `darwin-amd64/` |
| macOS (Apple Silicon) | `submit` از `darwin-arm64/` |

```bash
# در Tails/Linux:
chmod +x submit
```

### بررسی چک‌سام

```bash
sha256sum submit
# با فایل SHA256SUMS از مخاطبتان مقایسه کنید
```

## مرحله ۳: پیام را ارسال کنید

```bash
echo "عبارت-عبور-مخفی-شما" | ./submit \
  --relay https://xxxxxxxxxxxxxxxx.onion \
  --recipient-key abcdef0123456789abcdef0123456789abcdef0123456789abcdef0123456789 \
  --message "پیام شما اینجا"
```

خواهید دید:
```
Passphrase:
Submitted successfully.
```

**تمام شد.** پیام رمزنگاری و ارسال شده است. برنامه هیچ فایلی از خود به جا نگذاشته است.

---

## آنچه در پس‌زمینه رخ می‌دهد

1. از عبارت شما یک کلید رمزنگاری استخراج می‌شود (۲۸ ثانیه محاسبه -- این برای محافظت است)
2. پیام با کلید گیرنده رمزنگاری می‌شود (هیچ‌کس دیگری نمی‌تواند آن را بخواند)
3. پیام به اندازه استاندارد تکمیل می‌شود (تا قابل تشخیص نباشد)
4. به یک «صندوق پستی مرده» روی سرور رله ارسال می‌شود
5. گیرنده پیام را از همان صندوق برمی‌دارد (همان عبارت را می‌داند)
6. تمام کلیدها و داده‌ها از حافظه پاک می‌شوند
7. برنامه خاتمه می‌یابد

---

## قوانین امنیتی

### انجام دهید

- از Tails یا حداقل مرورگر Tor استفاده کنید
- از WiFi عمومی ارسال کنید (کافه، کتابخانه)
- عبارت را حفظ کنید -- هیچ‌جا ننویسید
- پس از ارسال برنامه را حذف کنید (`rm submit`)
- هر پیام را از مکان متفاوتی ارسال کنید (در صورت امکان)

### انجام ندهید

- از WiFi خانه ارسال نکنید
- از تلفن با سیم‌کارت به نام خودتان استفاده نکنید
- عبارت را در تلفن، یادداشت‌ها یا فضای ابری ذخیره نکنید
- برنامه را بیش از حد لازم روی کامپیوتر نگه ندارید
- به کسی نگویید که از این سیستم استفاده می‌کنید
- پیام یکسان را از چند کانال ارسال نکنید (همبستگی)

---

## وضعیت اجبار (duress)

اگر کسی شما را مجبور به نشان‌دادن سیستم کند:

1. **عبارت دیگری** وارد کنید (نه عبارت واقعی)
2. سیستم نشان می‌دهد: "Authentication failed. Please try again."
3. تمام داده‌ها به‌طور خودکار از حافظه پاک می‌شوند
4. برای ناظر مانند اشتباه تایپی به نظر می‌رسد -- قابل تشخیص از اجبار نیست

---

## سوالات متداول

**ارسال چقدر طول می‌کشد؟**
حدود ۳۰ ثانیه (بیشتر آن استخراج کلید از عبارت است -- عمدی است).

**اگر عبارت اشتباه وارد کنم چه می‌شود؟**
نشان می‌دهد "Authentication failed." چیزی ارسال نمی‌شود، چیزی ذخیره نمی‌شود.

**آیا اپراتور رله می‌تواند پیام را بخواند؟**
خیر. پیام با کلید گیرنده رمزنگاری شده است. رله فقط یک بلوب رمزنگاری‌شده می‌بیند.

**آیا کسی می‌تواند بفهمد به چه کسی می‌نویسم؟**
رله نمی‌داند شما کی هستید (نام مستعار) و پیام کجا می‌رود (صندوق مرده).

**اگر رله مصادره شود چه؟**
پیام‌ها رمزنگاری شده‌اند. بدون کلید گیرنده قابل خواندن نیستند.

---

## برای گیرنده: دریافت پیام‌ها

گیرنده از فایل `receive` با همان عبارت مشترک استفاده می‌کند:

### نمایش کلید عمومی (یکبار، به اشتراک با فرستنده)

```bash
echo "عبارت-مشترک" | ./receive --show-key
```

### دریافت پیام‌ها

```bash
echo "عبارت-مشترک" | ./receive --relay https://xxxxxxxx.onion
```

### مهم

- پیام‌ها فقط **یکبار** قابل دریافت هستند -- پس از دریافت حذف می‌شوند
- عبارت را **حضوری** یا از طریق کانال رمزنگاری‌شده به اشتراک بگذارید
- کلید عمومی کافی‌ست یکبار به اشتراک گذاشته شود
